## Submission Checklist

### Pre-Submission

**Technical**:
- [ ] App builds without errors
- [ ] All features tested on real devices
- [ ] Crashes fixed (0 crash rate)
- [ ] Performance optimized (< 3s load time)
- [ ] Backend API production-ready
- [ ] Analytics integrated
- [ ] Crashlytics/Firebase configured

**Legal**:
- [ ] Privacy policy created and hosted
- [ ] Terms of service created and hosted
- [ ] Content rating completed
- [ ] Copyright/trademark verified

**Assets**:
- [ ] App icons (all sizes)
- [ ] Screenshots (required sizes)
- [ ] Feature graphic (Play Store)
- [ ] Promotional video (optional)
- [ ] All text proofread

**Accounts**:
- [ ] Google Play Developer account ($25)
- [ ] Apple Developer account ($99/year)
- [ ] Payment merchant account configured

### Google Play Store Submission

1. **Sign In**: https://play.google.com/console
2. **Create App**: Click "Create app"
3. **Store Listing**: Fill all required fields
4. **App Content**:
   - Complete questionnaire
   - Add privacy policy URL
   - Target audience & content
5. **Pricing & Distribution**:
   - Countries: India (primary)
   - Free with in-app purchases
6. **App Releases**:
   - Upload AAB file
   - Release notes
   - Rollout percentage
7. **Submit for Review**

**Review Time**: Typically 1-3 days

### Apple App Store Submission

1. **App Store Connect**: https://appstoreconnect.apple.com
2. **Create App Record**: Fill app information
3. **Pricing**: Free with in-app purchases
4. **App Information**:
   - Screenshots
   - Description
   - Keywords
   - Support URL
   - Marketing URL
5. **Build**: Upload via Xcode or Transporter
6. **App Review Information**:
   - Contact details
   - Demo account (if applicable)
   - Notes for reviewer
7. **Submit for Review**

**Review Time**: Typically 1-7 days

## App Review Guidelines

### Common Rejection Reasons

**Google Play**:
- Incomplete content rating
- Missing privacy policy
- Inaccurate screenshots
- Broken functionality
- Inappropriate content

**Apple App Store**:
- Crashes or bugs
- Incomplete information
- Poor performance
- Privacy violations
- Copycat design

### How to Avoid Rejection

1. **Test thoroughly**: No crashes, all features work
2. **Accurate description**: Don't oversell features
3. **Privacy compliance**: Clear data usage disclosure
4. **Performance**: Fast loading, responsive UI
5. **Complete metadata**: All fields filled correctly
6. **Follow guidelines**: Read platform-specific rules

### Review Response Template

If rejected, respond professionally:

```
Dear Review Team,

Thank you for your feedback on CallCraft (Version 1.0.0).

We have addressed the issues you mentioned:

1. [Issue 1]: [How you fixed it]
2. [Issue 2]: [How you fixed it]

We have uploaded a new build (Version 1.0.1) with these fixes.

We believe the app now meets all guidelines and provides value to users.

Please let us know if you need any additional information.

Best regards,
CallCraft Team
```
